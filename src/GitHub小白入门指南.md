# GitHub小白入门指南

> 专为零基础用户准备的超简单教程

## 什么是GitHub？

GitHub就像一个放代码的"网盘"，但功能更强大。你可以在这里：
- 保存自己的代码
- 看别人写的代码
- 给别人的项目提问题（这就是今天要学的"提交issue"）

---

## 第一步：注册GitHub账号（5分钟搞定）

### 1. 打开注册页面
- 在浏览器地址栏输入：`github.com`
- 按回车键打开

### 2. 填写注册信息
页面上会看到3个框框，按顺序填写：

```
┌─────────────────────────────┐
│  用户名（你想叫啥名字）      │  ← 随便起个名字，比如：xiaoming123
└─────────────────────────────┘
┌─────────────────────────────┐
│  邮箱（你的QQ邮箱就行）      │  ← 输入：123456@qq.com
└─────────────────────────────┘
┌─────────────────────────────┐
│  密码（想个复杂点的）        │  ← 至少8位，最好有字母+数字
└─────────────────────────────┘
```

### 3. 验证你不是机器人
- 页面上会出现一个"我不是机器人"的复选框
- 点击打勾✓
- 可能会让你选图片，按提示选就行

### 4. 邮箱验证
- 登录你的QQ邮箱
- 找一封来自GitHub的邮件（可能在垃圾箱）
- 点击邮件里的"验证邮箱"按钮

### 5. 完成注册
- 验证成功后，GitHub会问一些个人信息
- 不想填就直接点"跳过"或"Skip"

🎉 **恭喜！账号注册完成！**

---

## 第二步：找到你想提问的地方

### 情况1：你知道项目的网址
- 直接在浏览器地址栏输入项目地址
- 比如：https://github.com/用户名/项目名

### 情况2：你不知道具体地址
- 在GitHub页面最上方，有个搜索框
- 输入你要找的项目名称
- 比如输入："微信"，然后按回车
- 在搜索结果中点击你要的项目

---

## 第三步：提交你的第一个Issue（问题）

### 1. 找到"Issues"按钮
进入项目页面后，仔细看页面上方有一排按钮：

```
┌──┬──┬──┬──┬──┬──┐
│Code│Issues│Pull requests│Actions│Projects│Wiki│
└──┴──┴──┴──┴──┴──┘
```

**点击"Issues"**

### 2. 创建新Issue
- 进入Issues页面后，会看到一个大大的绿色按钮
- 上面写着"New issue"或"新问题"
- **点击这个绿色按钮**

### 3. 填写问题标题
在"Title"（标题）框里，简单描述你的问题：

❌ **错误示例：**
```
不好了，出问题了！
这个怎么用？
救命啊！
```

✅ **正确示例：**
```
登录按钮点击没有反应
安装后打不开软件
第3步操作报错：文件找不到
```

### 4. 填写问题详情
在大文本框里，按这个模板写：

```
## 我遇到了什么问题：
（在这里简单描述）

## 问题是怎么发生的：
1. 我先点了XXX
2. 然后又点了XXX
3. 突然就XXX了

## 我希望的结果：
（描述你希望正常应该是什么样）

## 我的环境：
- 电脑系统：Windows 10
- 浏览器：QQ浏览器
- 其他相关信息：
```

### 5. 提交Issue
- 检查一遍写的内容
- 点击页面下方的绿色按钮"Submit new issue"
- 提交成功后，页面会跳转，地址栏会显示类似：`github.com/用户名/项目/issues/123`

🎉 **完成！你的Issue提交成功！**

---

## 第四步：后续要做什么

### 关注回复
- 你的邮箱会收到回复通知
- 或者回到GitHub，在页面右上角会显示小红点

### 如何回复别人
- 进入你的Issue页面
- 在最下面的文本框里输入你想说的话
- 点击绿色的"Comment"按钮发送

### 关闭Issue
当问题解决后，你可以关闭它：
- 在你的Issue页面
- 找到"Close issue"按钮（通常在页面下方）
- 点击即可关闭

---

## 超重要提醒

### ❌ 千万不要做：
1. **不要重复发相同的Issue** - 先搜索有没有类似问题
2. **不要发无意义的内容** - 比如"顶"、"同问"、"有人吗"
3. **不要使用不礼貌的语言** - 大家都是志愿者
4. **不要催得太急** - 维护者有自己的工作，需要时间

### ✅ 一定要做好：
1. **先搜索再提问** - 很多问题别人已经问过了
2. **提供详细信息** - 越详细越容易得到帮助
3. **保持礼貌** - 说"请"、"谢谢"不会吃亏
4. **有耐心** - 好问题值得等待好答案

## 常见问题（小白专用版）

**Q: 我英文不好怎么办？**
A: 没关系！现在很多中国开发者，直接用中文提问就行。如果项目是国外的，可以用简单英文，或者先用中文，下面用Google翻译一份英文。

**Q: 多久会有人回复我？**
A: 这个说不准，有的几分钟，有的几个月。热门项目回复快，冷门项目可能要等。

**Q: 没有人回复我怎么办？**
A: 
- 先检查你的问题描述是否清楚
- 补充更多详细信息
- 礼貌地询问是否需要提供其他信息

**Q: 我可以删除我发的Issue吗？**
A: 普通用户删不了，但可以关闭。如果真的需要删除，可以联系项目管理员。

**Q: 发错了怎么办？**
A: 可以在下面评论说明情况，然后关闭这个Issue，重新发一个正确的。
